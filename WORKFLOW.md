# 웹사이트 자동사냥 워크플로우

## 단계 개요

```
[1. 기획서] → [2. 평가] ─✗→ 피드백 → [1]
                  ↓✓
             [3. 설계서] → [4. 평가] ─✗→ 피드백 → [3]
                               ↓✓
                      [5. 스켈레톤] → [6. 평가] ─✗→ 피드백 → [5]
                                          ↓✓
                                    [7. 코드 작성] → [7. 평가] ─✗→ git revert + 피드백 → [7]
                                                         ↓✓
                                                    [8. 검토]
                                                         ↓
                                                    [9. 배포]
```

---

## 단계별 정의

### 1. 기획서 작성 (Claude)

**입력:** 사용자 요구사항 (자유 형식)
**출력:** `docs/01_기획서.md`
**포함 항목:**
- 사이트 목적 및 타겟 사용자
- 핵심 기능 목록 (MoSCoW 우선순위)
- 기술 스택 제안 및 근거
- 페이지 구성 (사이트맵)
- 레퍼런스 및 디자인 방향

---

### 2. 기획서 평가 (사용자)

**방법:** `docs/01_기획서.md` 직접 검토
**승인:** "승인" → 3단계 진행
**거부:** `docs/feedback/기획서_피드백_N.md` 작성 후 Claude에게 전달 → 1단계 재시작

---

### 3. 설계서 작성 (Claude)

**입력:** 승인된 기획서
**출력:** `docs/02_설계서.md`
**포함 항목:**
- 컴포넌트 구조도
- 페이지별 레이아웃 명세
- 색상 시스템 / 타이포그래피
- 데이터 구조 (필요 시)
- API 엔드포인트 (필요 시)
- 반응형 브레이크포인트

---

### 4. 설계서 평가 (사용자)

**방법:** `docs/02_설계서.md` 직접 검토
**승인:** "승인" → 5단계 진행
**거부:** `docs/feedback/설계서_피드백_N.md` 작성 후 Claude에게 전달 → 3단계 재시작

---

### 5. 코드 스켈레톤 구성 (Claude)

**입력:** 승인된 설계서
**출력:** `docs/03_스켈레톤.md` + 실제 빈 파일들 생성
**포함 항목:**
- 전체 파일/디렉토리 구조
- 각 파일의 책임 (단 한 줄 설명)
- 모든 클래스/함수 시그니처 (구현부 없음, `pass` 또는 stub만)
- import 목록

---

### 6. 스켈레톤 평가 (사용자)

**방법:** `docs/03_스켈레톤.md` + 생성된 파일 구조 검토
**승인:** "승인" → 7단계 진행
**거부:** `docs/feedback/스켈레톤_피드백_N.md` 작성 후 Claude에게 전달 → 5단계 재시작

---

### 7. 코드 작성 (Claude) + 평가 루프

**입력:** 승인된 스켈레톤
**방식:** 파일 단위 또는 기능 단위로 나눠서 구현, 각 단위마다 커밋

**각 단위 평가:**
- **승인:** 다음 단위 진행
- **거부:**
  1. 사용자가 `git revert <commit>` 또는 직접 수정
  2. `docs/feedback/코드_피드백_N.md` 에 문제점 기술
  3. Claude가 피드백 반영 후 재구현

**완료 기준:** 모든 기능 단위 승인 → 8단계 진행

---

### 8. 검토 (Claude + 사용자)

**Claude 작업:**
- 전체 코드 일관성 검사
- 누락 기능 체크 (기획서 대비)
- 기본 성능/접근성 체크

**사용자 작업:**
- 브라우저에서 실제 동작 확인
- 최종 수정 요청 (있을 경우 7단계 루프)

---

### 9. 배포 (Claude + 사용자 승인)

**Claude 작업:** 배포 설정 파일 생성, 배포 명령 제안
**사용자 작업:** 배포 명령 직접 실행 (승인 후)

---

## 피드백 파일 형식

`docs/feedback/[단계]_피드백_N.md`

```markdown
## 피드백 #N — [단계명]
날짜: YYYY-MM-DD

### 문제점
- ...

### 원하는 방향
- ...

### 참고 사항
- ...
```
